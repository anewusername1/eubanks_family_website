Content

Just plain HTML without <HTML> <HEAD> <BODY> tags. Note that you'd better make the markup style consistent with DTD of index.html, which is xhtml1-strict in this release.

The URL of a module can point to any static HTML file or dynamic content generated by server side scripts.

Styles

Try to keep the content clear of CSS code, move it into style.css and use class attributes instead.

To find out how to customize the outter look & feel of a module, see Module Tab

Behavior

Minimize the use of inline JavaScript, except for very specific one-off situations.

If you want certain controls in a module, embed control-specific HTML segment with proper class attributes set. For example, to use a tabs control, simply insert following code:

<div class="tabs">
	<ul class="tabsul">
		<li>tab1</li>
		<li>tab1</li>
		<li>tab1</li>
	</ul>
	<div>Content 1</div>
	<div>Content 2</div>
	<div>Content 3</div>
</div>

And you'll get a control as below:

  • tab1
  • tab1

Content 2
Content 3

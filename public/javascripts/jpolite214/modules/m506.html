A Tale of Two JS Files

Two JavaScript files are provide to build a working JPolite system:

  • jpolite.core.js - the abstract foundation of a tab-navigated portal framework, with internal data structures and methods.
  • jpolite.ext.js - all the feature initialization and customization code.

Normally you don't need to touch the core, customizing the ext part is (hopefully) enough to tailor JPolite for your needs.

Feature Customization

Here's a brief list of features you can customize in jpolite.ext.js:

  • Custom jQuery Plugins - you can simple add them via $.fn.extend.
  • Live Event Handlers - Simply call jquery function live to register you custom live event handlers. You can put all such calls in function myLiveEvents for easier management.
  • Custom Events - Simply call $.bindEvent to register a series of custom events and handlers into the JPolite system. Note that $.bindEvent is a JPolite method which is used to register global events not applicable to specific DOM elements. For DOM elements, please use jQuery's $.bind method instead. You can put all such calls in function myCustomEvents.
  • Message Handlers - Message handlers are used to process server side messages sent in JSON format. This shall be a convenient mechanism to build custom Ajax applications. More details can be found in Messaging Tab
  • Custom Controls - You need to register the selectors and initialization methods with arguments to make all control-specific HTML segments dynamically loaded with modules into active controls. Simply add a line in function myControls and Voila, your custom controls are activated. Further instructions can be found in module 406.
  • Navigation Tab - Last not least, the navigation tab is also customizable, further instructions in module 501.

JPolite2 Initialization Options

Here's a list of init arguments you can set when calling $.jpolite.init:

  • cts - Navigation Tab container id, default = "#main_nav"
  • its - Navigation Tab item selector, e.g., "dt" for a DL, default = "li"
  • t1 - Content transition Out callback when traversing between tabs, default = $.fn.fadeOut
  • t2 - Content transition In callback, default = $.fn.fadeIn
  • navInit - Navigation Tab Initialization callback, default = TraditionalTabs
  • navInitArguments - Navigation Tab Initialization parameters, default = {}
  • moduleSortable - Whether to allow module drag-n-drop, default = true
  • layoutPersistence - Methods to load/save layout of modules, default = [none, none]

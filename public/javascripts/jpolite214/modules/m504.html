Page background under a certain tab can be customized in _columnLayout variable in modules.js, e.g.,

	...
	_default: { bg:'normal',
	...
	},
	t2:{ bg:'darker',
	...
	},
	...

The value 'normal' and 'darker' refers to two class names defined in styles.css. That is, you customize page background through the use of CSS classes, which can use both colors and images

Known Issues

Currently JPolite2 makes use of jQuery UI's switchClass function to make a smooth transition between different background settings. But this feature does not work under WebKit based browsers, i.e., Google Chrome & Apple Safari. Even worse, the background stay unchanged between tabs.

So far there seems no sound solution yet that you need to make a decision here:

  1. Forget about transition, use plain attr("class", "new_class_name") to achieve maximum compatibility.
  2. Forget about WebKit, make switchClass a special treat for non-WebKit browsers.
